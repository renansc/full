Guia do Zap Workflow

Use esta pagina como referencia rapida para rodar o projeto, integrar WhatsApp, conectar a agenda e homologar a operacao em producao.

Visao geral Fluxo de atendimento Instalacao Integracoes Producao

Visao geral

  • Login por usuario com sessao Flask-Login.
  • Kanban com estados, departamentos e etiquetas personalizaveis.
  • Cadastro rapido de cliente, servico e historico de conversa.
  • Mensagem de WhatsApp identificando quem falou: cliente ou operador.
  • Tickets novos entram em Aguardando e podem trocar de departamento.
  • Cards concluidos ha mais de 2 dias saem da tela automaticamente.
  • Agenda compartilhada via Google Sheets e lembretes programados.

Fluxo de atendimento

Instalacao local

python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
python -m flask --app wsgi:app init-db
python -m flask --app wsgi:app run

O banco pode ser MySQL ou PostgreSQL via DATABASE_URL.

Integracoes sem custo inicial

  1. Crie o app na Meta e habilite o WhatsApp Cloud API.
  2. Copie o Phone Number ID, Access Token e Verify Token.
  3. Use Google Sheets para a agenda compartilhada.
  4. Se preferir validar localmente, exponha o webhook com ngrok e configure PUBLIC_BASE_URL.
  5. Preencha as variaveis de ambiente no menu Configuracao.

Webhook com ngrok

  1. Rode o app localmente na porta 5000.
  2. Abra um tunnel com ngrok http 5000.
  3. Copie a URL publica exibida pelo ngrok.
  4. Defina PUBLIC_BASE_URL com essa URL no .env.
  5. Use a callback completa /webhooks/whatsapp.

Homologacao e producao

  1. Suba em homologacao usando o banco de testes.
  2. Revise os estados, departamentos e usuarios no menu Configuracao.
  3. Teste o webhook do WhatsApp com mensagem real e confira o ticket novo.
  4. Teste a agenda e os lembretes antes de liberar para a equipe.
  5. Quando validar, publique a revisao no GitHub e faca deploy da mesma versao.

Endpoints principais

  • GET / - kanban principal.
  • GET /docs - documentacao no menu.
  • GET /calendar - tela da agenda.
  • GET /api/dashboard - dados do kanban e departamentos.
  • GET /api/integrations/status - status das integracoes.
  • GET /webhooks/whatsapp e POST /webhooks/whatsapp - webhook WhatsApp.
  • POST /api/whatsapp/send - envio manual de mensagem.
