Guia rapido do GPS Musical

Use esta aba para orientar cadastro, audio, modo palco, sincronizacao e publicacao do repertorio sem sair do app.

Visao geral Fluxo de uso Fontes de audio Configuracao Sync e backup Publicacao

Visao geral

  • Biblioteca central com busca, ordenacao e abertura rapida do repertorio.
  • Editor por blocos para organizar cifra, letra, titulo auxiliar e tempo.
  • Fonte de audio por musica com suporte a MP3 local, YouTube e Spotify.
  • Modo Palco com navegacao por bloco, progresso e timeline sincronizada.
  • Sincronizacao remota do repertorio e criacao de snapshots no banco atual.

Fontes de audio

  • MP3 local: fica salvo apenas neste navegador via IndexedDB. Ideal para playback proprio.
  • YouTube: aceita link ou ID e permite marcar tempos automaticamente na timeline.
  • Spotify: incorpora track, album, playlist ou episode para referencia rapida.
  • Quando uma musica usa MP3 local, o JSON e o banco remoto salvam so os metadados, nao o arquivo binario.

Modo Palco

  1. Abra uma musica na biblioteca e entre na tela de visualizacao.
  2. Valide a fonte de audio e, se quiser, marque os tempos bloco a bloco.
  3. Use Play timeline para seguir os tempos marcados.
  4. Use Play slide quando preferir avancar blocos por intervalo fixo.
  5. No palco, navegue com as setas do teclado e feche com Esc.

Configuracao e integracoes

  • O menu Config concentra banco remoto, YouTube e Spotify.
  • Para Alwaysdata, normalmente use MySQL com host, porta, banco, usuario e senha.
  • YouTube pede API Key, Client ID e Redirect URI para autenticar no navegador.
  • Spotify usa Client ID e Redirect URI para o fluxo PKCE; o Client Secret fica opcional.
  • Depois de salvar, use Testar conexao para validar o banco antes de sincronizar.

Sincronizacao e backup

  1. A aba Biblioteca mostra o status do backend e permite sincronizar manualmente.
  2. A aba Backup exporta e importa JSON do repertorio local.
  3. O botao Criar backup remoto gera snapshots no banco configurado para o GPS Musical.
  4. Ao importar um JSON, os MP3 locais precisam ser vinculados novamente neste navegador.

Publicacao e operacao

  • Revise o repertorio na biblioteca antes de publicar uma nova versao.
  • Confirme se a conexao remota esta valida e se os backups aparecem na lista.
  • Teste pelo menos uma musica com YouTube, uma com Spotify e uma com MP3 local.
  • Quando estiver homologado, envie a revisao ao GitHub e publique a mesma versao no servidor.

Rotas e armazenamento

  • GET /gpsmusical/ - interface principal do app.
  • GET /api/gps/config - leitura e gravacao da configuracao do GPS Musical.
  • POST /api/gps/config/test-database - valida a conexao do banco configurado.
  • GET /api/gps/backups e POST /api/gps/backups - lista e cria backups remotos.
  • localStorage guarda o repertorio local e IndexedDB guarda os blobs de MP3.
